# Учебный Django-проект

Простой клиент-серверный веб-проект на Django, выполненный в рамках практической работы.

## Стек
- Python
- Django
- HTML
- CSS
- SQLite

## Структура веток
- `main` — стабильная версия проекта
- `dev` — ветка для интеграции
- `feature/*` — индивидуальные ветки участников

## Правила работы
1. Каждый участник работает только в своей ветке `feature/*`.
2. Все изменения сначала вливаются в `dev`.
3. В `main` попадает только проверенная и рабочая версия.
4. Каждый Pull Request должен иметь понятное описание изменений.

## Запуск проекта

### 1. Перейти в папку project
```bash
cd project
```

### 2. Создать и активировать виртуальное окружение

Linux/macOS:

```bash
python3 -m venv .venv
source .venv/bin/activate
```

Windows PowerShell:

```bash
python -m venv .venv
.venv\Scripts\Activate.ps1
```

### 3. Установить зависимости
```bash
pip install -r ..\requirements.txt
```

### 4. Выполнить миграции
```bash
python manage.py migrate
```

### 5. Запустить сервер
```bash
python manage.py runserver
```

## Доступные страницы
- `/` — главная
- `/login/` — вход
- `/items/` — объекты
- `/categories/` — категории
- `/admin/` — админка

Админ django не создан.

## Как войти
1. Перейдите в папку `project`.
2. Создайте администратора:
```bash
python manage.py createsuperuser
```
3. Запустите проект и откройте `/login/`.
4. Введите логин и пароль созданного superuser.

## Что отвечает человек 5
- подключение Django auth
- настройка `login` и `logout`
- ограничение доступа к действиям создания, редактирования и удаления
- отображение разных кнопок в шаблонах для гостя и авторизованного пользователя
- подготовка чеклиста и сценария демонстрации

## Ограничения доступа
- Просмотр списков и страниц можно оставить доступным всем.
- Создание, редактирование и удаление доступны только авторизованным пользователям.
- Выход выполняется POST-запросом на `/logout/`.
